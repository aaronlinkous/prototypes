How It Works:

  • Grabs desktop form values on load and sets our inputs to that value
  • If the desktop's inputs are empty, and we set a value in ours, it will auto populate the desktop fields.
  • When we update our inputs by typing, pasting, etc it will update the desktop
  • We can override the desktops default value on initial load with a class of .override on our inputs
  • Data id's on both form and inputs assign inputs to the desktop's form and input id. Form data-id's are used because desktop id's are sometimes not unique across multiple forms.
